## 概述

对象存储接口工作流提供了一套统一的文件管理能力，支持文件上传、文件信息查询和文件替换功能。所有接口通过统一的入口访问，使用标准化的请求和响应格式。

**版本**: v1.0.0  
**更新时间**: 2026-04-27

---

## 统一入口

### 请求格式

所有接口通过统一入口访问，请求格式如下：

```json
{
  "action": "接口名称",
  "params": "接口参数JSON字符串"
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | 是 | 接口名称，可选值：`upload`、`get_file`、`upload_and_replace` |
| params | string | 是 | 接口参数的JSON字符串，具体参数见各接口说明 |

### 通用响应格式

所有接口的响应格式统一如下：

```json
{
  "success": true,
  "action": "接口名称",
  "data": {
    // 接口特定数据
  },
  "message": "操作结果描述"
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作是否成功 |
| action | string | 接口名称 |
| data | object | 接口返回的具体数据，结构因接口而异 |
| message | string | 操作结果描述 |

### 错误响应格式

当操作失败时，响应格式如下：

```json
{
  "success": false,
  "action": "接口名称",
  "error": "错误描述",
  "message": "错误信息"
}
```

## 接口列表

### 1. upload - 上传文件接口

#### 功能说明

从指定URL下载文件并上传到对象存储，返回文件key。

#### 请求参数

**params 字段示例**：

```json
{
  "upload_url": "https://example.com/file.jpg"
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| upload_url | string | 是 | 要上传的文件URL，支持HTTP/HTTPS协议 |

#### 响应数据

**成功响应示例**：

```json
{
  "success": true,
  "action": "upload",
  "data": {
    "file_key": "example_abc123.jpg"
  },
  "message": "文件上传成功"
}
```

**data 字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| file_key | string | 上传成功后的文件key，可用于后续操作 |

#### 完整请求示例

```json
{
  "action": "upload",
  "params": "{\"upload_url\": \"https://example.com/image.jpg\"}"
}
```

#### 完整响应示例

```json
{
  "success": true,
  "action": "upload",
  "data": {
    "file_key": "image_a1b2c3d4.jpg"
  },
  "message": "文件上传成功"
}
```

### 2. get_file - 获取文件信息接口

#### 功能说明

通过file_key获取文件详细信息，包含24小时有效访问URL。

#### 请求参数

**params 字段示例**：

```json
{
  "file_key": "example_abc123.jpg"
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file_key | string | 是 | 要查询的文件key |

#### 响应数据

**成功响应示例**：

```json
{
  "success": true,
  "action": "get_file",
  "data": {
    "file_key": "example_abc123.jpg",
    "file_url": "https://storage.example.com/example_abc123.jpg?sign=...",
    "expire_time": 86400,
    "expire_description": "24小时"
  },
  "message": "文件信息获取成功"
}
```

**data 字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| file_key | string | 文件key |
| file_url | string | 文件访问URL，有效期24小时 |
| expire_time | number | URL有效期（秒） |
| expire_description | string | 有效期描述 |

#### 完整请求示例

```json
{
  "action": "get_file",
  "params": "{\"file_key\": \"image_a1b2c3d4.jpg\"}"
}
```

#### 完整响应示例

```json
{
  "success": true,
  "action": "get_file",
  "data": {
    "file_key": "image_a1b2c3d4.jpg",
    "file_url": "https://coze-coding-project.tos.coze.site/coze_storage_7633312193307541539/image_a1b2c3d4.jpg?sign=1777356087-bb78d75d49-0-1d120ff5459b62dc88c838d2068c837db6769cd2920d965fc3e48a66e7b7bedd",
    "expire_time": 86400,
    "expire_description": "24小时"
  },
  "message": "文件信息获取成功"
}
```

### 3. upload_and_replace - 上传并替换文件接口

#### 功能说明

上传新文件，删除旧文件，返回新文件的key。适用于文件更新场景。

#### 请求参数

**params 字段示例**：

```json
{
  "upload_url": "https://example.com/new_file.jpg",
  "file_key": "example_abc123.jpg"
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| upload_url | string | 是 | 新文件的URL |
| file_key | string | 是 | 要删除的旧文件key |

#### 响应数据

**成功响应示例**：

```json
{
  "success": true,
  "action": "upload_and_replace",
  "data": {
    "new_file_key": "newfile_def456.jpg"
  },
  "message": "文件替换成功"
}
```

**data 字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| new_file_key | string | 新上传文件的key |

#### 执行流程

1. 从 `upload_url` 下载文件并上传到对象存储
2. 删除 `file_key` 指定的旧文件
3. 返回新上传文件的key

**注意**：即使旧文件删除失败，新文件也会成功上传并返回。

#### 完整请求示例

```json
{
  "action": "upload_and_replace",
  "params": "{\"upload_url\": \"https://example.com/new_image.jpg\", \"file_key\": \"image_a1b2c3d4.jpg\"}"
}
```

#### 完整响应示例

```json
{
  "success": true,
  "action": "upload_and_replace",
  "data": {
    "new_file_key": "newimage_e5f6g7h8.jpg"
  },
  "message": "文件替换成功"
}
```

## 错误码说明

### 常见错误类型

| 错误类型 | 说明 | 解决方案 |
|---------|------|---------|
| 参数格式错误 | params字段不是有效的JSON字符串 | 检查params字段是否为合法的JSON |
| 文件不存在 | 指定的file_key对应的文件不存在 | 确认file_key是否正确 |
| 文件上传失败 | 从URL下载或上传文件失败 | 检查upload_url是否可访问，网络是否正常 |
| 接口类型错误 | action参数值不在允许的范围内 | 确认action参数为upload、get_file或upload_and_replace之一 |

### 错误响应示例

**参数格式错误**：

```json
{
  "success": false,
  "action": "upload",
  "error": "参数格式错误，params必须是有效的JSON字符串",
  "message": "接口类型错误"
}
```

**文件不存在**：

```json
{
  "success": false,
  "action": "get_file",
  "error": "文件不存在: invalid_key.jpg",
  "message": "文件信息获取失败"
}
```

## 注意事项

### 1. URL格式要求

- `upload_url` 必须是完整的HTTP或HTTPS URL
- URL必须指向可公开访问的文件
- 支持的文件类型：图片（jpg、png、gif等）、视频、音频、文档等

### 2. 文件key说明

- 文件key是文件在对象存储中的唯一标识
- 文件key与上传时指定的文件名可能不同（系统会自动添加前缀）
- 请妥善保管文件key，后续所有操作都需要用到

### 3. URL有效期

- `get_file` 接口返回的 `file_url` 有效期为24小时
- 过期后需要重新调用 `get_file` 接口获取新的URL
- 建议在URL过期前及时更新

### 4. 文件替换注意事项

- `upload_and_replace` 接口中，即使旧文件删除失败，新文件也会成功上传
- 如果需要确保原子性，建议先调用upload接口，确认成功后再调用其他接口
- 删除旧文件的失败不会影响整体操作的返回结果

### 5. 请求大小限制

- `params` 字段的JSON字符串大小建议不超过10KB
- 单个文件大小建议不超过100MB

### 6. 并发限制

- 建议单用户并发请求数不超过10个
- 超过限制可能会导致请求失败或响应变慢
